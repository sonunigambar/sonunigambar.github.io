AWS VPC & EC2 Setup Guide

Complete step-by-step instructions for creating a public EC2 instance with Docker

  1. Go to AWS Console → VPC → Your VPCs → Create VPC
  2. Fill in the details:
    • Name: MyVPC
    • IPv4 CIDR block: 10.0.0.0/16 (supports 65,536 addresses)
    • IPv6 CIDR block: None (optional)
    • Tenancy: Default
  3. Click Create VPC

  1. Go to VPC → Subnets → Create subnet
  2. Fill in the details:
    • Name: PublicSubnet1
    • VPC: MyVPC
    • Availability Zone: e.g., ap-southeast-2a
    • IPv4 CIDR block: 10.0.1.0/24 (256 addresses)
  3. Click Create subnet

  1. Go to VPC → Internet Gateways → Create internet gateway
  2. Name it: MyIGW
  3. Attach it to your VPC:
    • After creation, select the IGW → Actions → Attach to VPC
    • In the pop-up, choose your VPC (MyVPC) → Click Attach internet gateway

  1. Go to VPC → Route Tables → Create route table
  2. Fill in:
    • Name: PublicRouteTable
    • VPC: MyVPC
  3. Select the route table → Edit routes → Add route:
    • Destination: 0.0.0.0/0
    • Target: MyIGW
  4. Associate the route table with the public subnet:
    • Subnet Associations → Edit subnet associations → select PublicSubnet1 → Save

  1. Go to EC2 → Launch Instance
  2. Select AMI & instance type
  3. Network: MyVPC
  4. Subnet: PublicSubnet1
  5. Auto-assign Public IP: Enabled (important!)
  6. Launch instance
Inbound Rules for EC2 Security Group
  • SSH (Port 22) → Your IP (recommended)
  • HTTP (Port 80) → 0.0.0.0/0 (anyone can access website)
  • HTTPS (Port 443) → 0.0.0.0/0 (if using SSL)
  • Custom TCP (Port 8080 for Spring Boot) → 0.0.0.0/0
  • Database Ports (MySQL 3306 / Postgres 5432) → restrict to security group / private network (do NOT expose publicly)
