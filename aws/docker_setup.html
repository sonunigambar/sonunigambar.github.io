EC2 Docker Setup Guide

Use SSH to connect to your EC2 instance:

ssh -i ec2-key-pair2.pem ec2-user@16.176.138.253

Explanation: This command connects you to your EC2 instance using the key pair ec2-key-pair2.pem. Replace the IP with your instance's public IP.

Install Docker on your EC2 instance:

sudo dnf install -y docker

Explanation: This command installs Docker using the DNF package manager on Amazon Linux 2 or other Fedora-based distributions. The -y flag automatically confirms installation prompts.

sudo systemctl start docker
sudo systemctl enable docker

Explanation: The first command starts the Docker service immediately. The second command ensures Docker starts automatically on system reboot.

sudo usermod -aG docker $USER
newgrp docker

Explanation: This allows your current user to run Docker commands without sudo. newgrp docker applies the group change immediately.

sudo docker ps

Explanation: Lists all running containers. Initially, it should be empty but confirms Docker is running correctly.

docker logs -f portfolio-controller

Explanation: Streams logs of the container portfolio-controller in real-time to monitor application output.

docker stop portfolio-controller

Explanation: Stops the running container named portfolio-controller.

docker run -d --name portfolio-controller -p 80:8082 portfolio-controller:latest

Explanation: Runs a new container named portfolio-controller in detached mode (-d), mapping host port 80 to container port 8082, using the portfolio-controller:latest image.
