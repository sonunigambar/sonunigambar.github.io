Docker Image Guide for Spring Boot

Build a lightweight, portable image for your Spring Boot app and understand each Dockerfile instruction. 

# Base JDK image
FROM eclipse-temurin:17-jdk-alpine

# Set working directory
WORKDIR /app

# Copy the jar file
COPY target/*.jar app.jar

# Expose Spring Boot default port
EXPOSE 8080

# Run the jar
ENTRYPOINT ["java", "-jar", "app.jar"]

Tip: Place this file in your project root as Dockerfile (no extension).

FROM eclipse-temurin:17-jdk-alpine

Uses a small, secure Alpine-based JDK 17 image—perfect for running Spring Boot.

WORKDIR /app

Sets the working directory inside the container to /app. If it doesn’t exist, Docker creates it. All next commands run from here.

COPY target/*.jar app.jar

Copies your built JAR from your host’s target/ folder into the image at /app/app.jar. Make sure you’ve built your project first (Maven/Gradle).

EXPOSE 8080

Documents that the app listens on port 8080. (It doesn’t publish the port; you map it at run.)

ENTRYPOINT ["java", "-jar", "app.jar"]

Defines the container’s start command—runs your Spring Boot app when the container starts.

1) Build your app JAR (Maven example):

mvn clean package -DskipTests

2) Build the Docker image:

docker build -t portfolio-controller:latest .

Explanation: -t tags the image name:version. The final dot means “use current folder as build context”.

Run locally (map host 8080 → container 8080):

docker run -d --name portfolio-controller -p 8080:8080 portfolio-controller:latest

Check it’s running:

docker ps

Follow logs:

docker logs -f portfolio-controller

Confirm the working directory exists:

docker exec -it portfolio-controller sh
ls -l /app
exit

Stop / Remove:

docker stop portfolio-controller
docker rm portfolio-controller

Login to ECR:

aws ecr get-login-password --region ap-southeast-2 \
| docker login --username AWS --password-stdin <account_id>.dkr.ecr.ap-southeast-2.amazonaws.com

Tag & Push:

docker tag portfolio-controller:latest <account_id>.dkr.ecr.ap-southeast-2.amazonaws.com/codingcure/portfolio-service:latest
docker push <account_id>.dkr.ecr.ap-southeast-2.amazonaws.com/codingcure/portfolio-service:latest

  • Port not reachable? Ensure you map -p host:container correctly and security groups/firewall allow it.
  • Alpine has no bash: Use sh (e.g., docker exec -it <name> sh).
  • Jar not found? Make sure you built the JAR and the path matches target/*.jar.
  • Multi-stage build (smaller image)? Build in Maven/Gradle image, then copy only the final JAR into a JRE base.

Example health check: expose an actuator endpoint and verify with curl http://localhost:8080/actuator/health (if enabled).
