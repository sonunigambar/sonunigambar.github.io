What is Caching?

Caching is a technique used to store frequently accessed data temporarily in a faster storage layer so that future requests can be served quickly.

Client → Application → Cache → Database

Why We Use Caching

Fast Response

Returns data quickly from memory.

Less DB Load

Reduces unnecessary database hits.

Scalability

Handles high traffic efficiently.

Better UX

Improves user experience.

Real-Life Examples of Caching

E-Commerce Website

Suppose 10 lakh users open the same product page during a sale.

Without Cache: Every request hits Database With Cache: First request hits DB Remaining requests served from Cache

Example: Product details, offers, ratings, images.

YouTube / OTT Platform

Frequently watched videos are cached closer to users using CDN cache.

User opens Video ↓ Video fetched from nearby CDN Cache ↓ Faster streaming experience

Example: Thumbnails, trending videos, subtitles.

Google Search

Popular search results are cached because millions of users search the same keywords repeatedly.

Search Query ↓ Check Cached Results ↓ Return Faster Response

Example: Trending searches, suggestions, maps data.

Banking Application

Some data is cached carefully to reduce DB load.

Frequently Read Data: - Branch Details - IFSC Codes - Currency Rates

Note: Sensitive live balance data is usually NOT cached.

Food Delivery Apps

Restaurant lists and menus are cached because users request them repeatedly.

User Opens App ↓ Nearby Restaurants Loaded from Cache ↓ Fast UI Loading

Example: Menus, banners, restaurant ratings.

Weather Application

Weather APIs are expensive and rate limited, so responses are cached for a few minutes.

First API Call ↓ Weather API Response ↓ Store in Cache (5 mins) ↓ Next users get cached response

Saves API cost and improves speed.

Social Media Platforms

News feed and user profile data are cached for faster scrolling experience.

Open Feed ↓ Fetch Cached Posts ↓ Smooth User Experience

Example: Profile images, stories, reels metadata.

Online Gaming

Game configuration and leaderboard data are cached for quick access.

Player Opens Game ↓ Load Cached Configurations ↓ Reduce Server Processing

Example: Rankings, rewards, settings.

When We Use Caching

  • Frequently accessed data
  • Rarely changing data
  • Heavy database queries
  • External API responses
  • Product catalog data

Types of Caching

1. In-Memory Cache

Stored inside JVM memory (RAM).

spring.cache.type=simple

2. Distributed Cache

Shared across multiple servers.

  • Redis
  • Memcached

Step 1: Add Cache Dependency

Add Spring Boot Cache dependency inside pom.xml 


<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-cache</artifactId>
</dependency>

Step 2: Enable Simple In-Memory Cache

Add below property inside application.properties 


spring.cache.type=simple

This creates an in-memory cache inside JVM memory.

Step 3: Create Cache Configuration Class

Create a cache manager configuration. 


package com.codingcure.config;

import org.springframework.cache.CacheManager;
import org.springframework.cache.concurrent.ConcurrentMapCacheManager;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class CachingConfig {

    @Bean
    public CacheManager cacheManager() {
        return new ConcurrentMapCacheManager("fund");
    }

}

Step 4: Use @Cacheable

Fetch data from cache if already available. 


@Service
public class FundService {

    @Cacheable(
        value = "fund",
        key = "#fundId",
        condition = "#fundId > 1"
    )
    public Fund getFund(int fundId) {

        Fund fundInfo = null;

        Optional<Fund> fund =
                fundRepo.findById(fundId);

        if (fund.isPresent()) {
            fundInfo = fund.get();
        }

        return fundInfo;
    }
}

Step 5: Use @CachePut

Update database and cache together. 


@CachePut(
    value = "fund",
    key = "#fund.fundId"
)
public Fund updatefund(Fund fund) {

    Fund updatedFund =
            fundRepo.save(fund);

    return updatedFund;
}

Step 6: Use @CacheEvict

Remove data from cache when deleting records. 


@CacheEvict(
    value = "fund",
    key = "#fundId"
)
public String deletefund(int fundId) {

    String deletedMsg = null;

    Optional<Fund> existingFund =
            fundRepo.findById(fundId);

    if (existingFund.isPresent()) {

        fundRepo.deleteById(fundId);

        deletedMsg =
                "Deleted successfully of Id: "
                        + fundId;

    } else {

        deletedMsg =
                "fund id is not available";
    }

    return deletedMsg;
}

Redis Cache Implementation in Spring Boot

Redis is a distributed in-memory cache used in production-level microservices applications for high performance and scalability.

Step 1: Add Redis Dependencies

Add below dependencies inside pom.xml 


<dependency>
    <groupId>org.springframework.data</groupId>
    <artifactId>spring-data-redis</artifactId>
    <version>3.4.4</version>
</dependency>

<dependency>
    <groupId>redis.clients</groupId>
    <artifactId>jedis</artifactId>
    <version>5.2.0</version>
</dependency>

Step 2: Configure Redis Properties

Add below properties inside application.properties 


spring.cache.type=redis

spring.data.redis.host=localhost
spring.data.redis.port=6379
spring.data.redis.database=1
  • host → Redis server host
  • port → Redis default port
  • database=1 → Redis logical DB

Step 3: Create Redis Cache Configuration

Configure Redis Cache Manager with TTL and Serialization. 


@Configuration
public class CachingConfig {

    @Bean
    public RedisCacheManager redisCacheManager(
            RedisConnectionFactory redisConnectionFactory) {

        RedisCacheConfiguration redisCacheConfiguration =
                RedisCacheConfiguration.defaultCacheConfig()
                .entryTtl(Duration.ofSeconds(30))
                .disableCachingNullValues()
                .serializeValuesWith(
                        RedisSerializationContext
                        .SerializationPair
                        .fromSerializer(
                            new GenericJackson2JsonRedisSerializer()
                        )
                );

        return RedisCacheManager
                .builder(redisConnectionFactory)
                .cacheDefaults(redisCacheConfiguration)
                .build();
    }

    @Bean
    RedisConnectionFactory redisConnectionFactory() {
        return new JedisConnectionFactory();
    }

}

Step 4: Understanding Redis Configuration

entryTtl(Duration.ofSeconds(30))

Cache data automatically expires after 30 seconds.

disableCachingNullValues()

Prevents storing null values inside Redis cache.

GenericJackson2JsonRedisSerializer

Converts Java objects into JSON format before storing.

JedisConnectionFactory

Creates Redis connection using Jedis client.

Step 5: Use Cache Annotations

Same cache annotations work with Redis also. 


@Cacheable(
    value = "fund",
    key = "#fundId"
)
public Fund getFund(int fundId) {

    return fundRepo.findById(fundId).get();
}

Flow:

First Request ↓ Fetch From Database ↓ Store Inside Redis ↓ Return Response Second Request ↓ Fetch Directly From Redis

Step 8: Important Redis Commands

redis-cli

Opens Redis Command Line Interface.

keys *

Displays all keys available inside Redis database.

get "fund::1"

Fetches cached value for fundId = 1.

info keyspace

Displays Redis database statistics and key information.

config get databases

Shows total logical databases configured in Redis.

select 1

Switches Redis connection to database number 1.

ttl "fund::1"

Displays remaining expiration time of cache key.

del "fund::1"

Deletes specific cache key manually from Redis.

Why Redis is Better Than Simple Cache?
  • Shared across multiple microservices instances
  • Extremely fast in-memory operations
  • Supports TTL (automatic expiration)
  • Works even after application restart
  • Production-ready distributed caching solution
